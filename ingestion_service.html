1. Project overview

What it is: A FastAPI microservice that scans S3 for already-cleaned PDFs and runs a PageIndex pipeline (hosted LLM API + local PDF text extraction) over them in batches, persisting hierarchical nodes + job JSON in PostgreSQL.

Not implemented in this repo: direct file upload endpoint, cleaning service integration, vector embeddings, CSV ingestion, Celery/Redis workers, presigned S3 URLs for clients.

Upstream: An external pipeline writes cleaned PDFs to S3 under cleaned/{stem}/{doc_id}/; this service is then triggered via POST /api/ingestion/ingest/run (manually or by automation).

Downstream: AWS S3 (read-only here); PostgreSQL; PageIndex HTTP API (PAGEINDEX_API_BASE).

Boundary: All heavy PDF parsing + LLM calls live under app/pageindex/; orchestration in app/application/ingestion_service.py and app/services/pageindex_pipeline.py.
